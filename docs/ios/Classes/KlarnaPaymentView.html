KlarnaPaymentView

public final class KlarnaPaymentView : UIView

A UIView rendering a payment method category. It acts as an interface to methods relevant to the payment method category it’s rendering.

    Category of payment method loaded in this view.

    Informs whether this PaymentView should be presented to the customer. True by default. Once it becomes false, it will not become true again.

Lifecycle
  • init(clientToken:category:returnUrl:delegate:)

    Initializes the Klarna payment view

    Note

    Klarna payment view will be initialized with frame .zero, auto layout is the recommended way to manage the payment view.

    Note

    When a Klarna payment view is initialized, it will automatically call initialize(clientToken: String, returnUrl: String), upon success or failure, according delegate methods will be called.

    Declaration

    Swift

    public convenience init(clientToken: String,
                        category: String,
                        returnUrl: String,
                        delegate: KlarnaPaymentViewDelegate)

    Parameters

    clientToken

    Client token received from Klarna when creating the session.
    category

    Category of payment methods to be loaded.
    returnUrl

    Your apps custom URL scheme CFBundleURLSchemes.
    delegate

    A delegate that will receive events from this view.
  • init(coder:)

    Klarna payment view does not support initialization from storyboard or xib files.

    Note

    You should never need to initialize a KlarnaPaymentView yourself, Use KlarnaPaymets‘s factory method to make a new instance of KlarnaPaymentView.

    Declaration

    Swift
